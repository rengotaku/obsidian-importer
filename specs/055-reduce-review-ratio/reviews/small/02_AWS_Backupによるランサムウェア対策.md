## 圧縮後

### ランサムウェア対策の現状

現代のランサムウェア攻撃はバックアップデータ自体を標的にするため、「バックアップを取っていれば安心」という考えは通用しなくなっている。IPAの「情報セキュリティ10大脅威2025」でもランサムウェア被害は10年連続1位。

### 3-2-1-1-0ルール

Veeamが提唱するバックアップ戦略。
- **3**: データは3コピー保持
- **2**: 2種類以上の異なるメディアに保存
- **1**: 1つはオフサイト（別リージョン）に保管
- **1**: 1つはイミュータブル（改変不可）な状態で保管
- **0**: 定期的なリカバリテストでエラー0件確認

### AWSでの実現方法

| ルール | AWSでの実現方法 |
|--------|----------|
| 3コピー | 本番データ ＋ 東京Vault ＋ 大阪Vault |
| 2メディア | EBS/RDS + S3 |
| 1オフサイト | クロスリージョンコピー（東京→大阪） |
| 1イミュータブル | 論理的エアギャップボールト |
| 0エラー | AWS Backup復元テスト機能 |

### アカウント構成

- **本番アカウント**: アプリケーションを動かす場所
- **バックアップアカウント**: バックアップデータを保管する専用アカウント。SCPでバックアップ削除を禁止
- **リカバリアカウント**: 復旧用のクリーンなアカウント。IaCから再構築してリストア

### 論理的エアギャップボールト

バックアップデータがAWSサービス所有のアカウントに保存され、ユーザーのアカウントとは完全に分離される。

### OrganizationsのSCP

Organizations配下のアカウントに対して「これは絶対やっちゃダメ」というルールを強制するポリシー。SCPはIAMより上位の制限。
