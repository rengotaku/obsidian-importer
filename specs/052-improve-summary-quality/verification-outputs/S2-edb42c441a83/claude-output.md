# SRE/DevOps書籍の読み進め方

## 質問の背景
SRE/DevOpsのおすすめ書籍リストを見て、どのように読み進めるのが効果的か質問された。

## 提案した3つのルート

### 1. 王道ルート（体系的に学ぶ）
**Phase 1: DevOpsのマインドセット**
- The Phoenix Project（小説形式で読みやすい）
- The DevOps ハンドブック（実践ガイド）

**Phase 2: SREの基礎**
- SRE本（Google公式、基本概念）
- SREワークブック（実践的プラクティス）

**Phase 3: 深掘り**
- SLO本、チームトポロジー、セキュリティ関連など興味に応じて

### 2. 実践重視ルート（手を動かしながら）
1. Infrastructure as Code
2. システム運用アンチパターン（薄くて実践的）
3. SRE本 or SREの知識地図
4. LeanとDevOpsの科学（改善指標の考え方）

### 3. 最短ルート（時間がない場合）
1. SREの知識地図（薄い、全体像把握）
2. LeanとDevOpsの科学（エビデンスベース）
3. The Phoenix Project（息抜き）

## 重要な補足
- **SRE本は重い**ので、Phoenix ProjectやSREの知識地図で雰囲気を掴んでから挑むほうが挫折しにくい
- SRE本のWeb版は無料公開されており、気になる章だけつまみ食いも可能
- ユーザーはホームラボでインフラ実践中なので、実践重視ルートが適している可能性
